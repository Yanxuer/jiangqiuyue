# 江秋月 — 小白使用说明书

> 最后更新: 2026-06-30 | 涵盖 23 个工具完整用法及六阶段工程流程

---

## 一、江秋月是谁？

江秋月是一个**桌面 AI 编程助手**。她的核心能力是：

> 你开口说需求，她自动读代码 → 改代码 → 跑命令 → 查文档 → 多轮迭代，直到任务完成。

她不是聊天机器人，而是**真正能干活的智能体**。你不需要一步一步指挥，只要说清楚最终目标。

### 江秋月的工作流程

她遵循严格的软件工程六阶段流程，确保每一步都经得起推敲：

```
DEFINE ──→ PLAN ──→ BUILD ──→ VERIFY ──→ REVIEW ──→ SHIP
 明确需求    任务拆解    增量实现    测试验证    代码审查    交付完成
```

| 阶段 | 她做什么 | 你的角色 |
|------|---------|---------|
| DEFINE | 列出假设让你确认，浏览项目结构 | 纠正假设偏差 |
| PLAN | 分析依赖关系，按功能路径（非技术层）拆分任务 | 确认优先级 |
| BUILD | 每次只改一处，改完立即编译验证，小步快跑 | 观察进度 |
| VERIFY | 遇错即停，系统排查根因，修复后加回归测试 | 提供额外信息 |
| REVIEW | 五轴审查：正确性/可读性/架构/安全/性能 | 最终确认 |
| SHIP | 检查 7 项质量门禁，确保无遗留问题 | 验收交付 |

> 这套方法论源自 [agent-skills](https://github.com/addyosmani/agent-skills)——一套面向 AI 编程智能体的生产级工程技能集合。

## 二、基础用法：从"一句指令"开始

### 2.1 编程任务（最强能力）

直接把需求丢给她：

```
"帮我修复 src/utils.js 第 42 行的空指针 bug"
"在项目里新增一个 /api/health 健康检查端点"
"把 README.md 里的 DeepSeek 配置全部换成通用的 LLM 配置"
```

她会自动：
1. 用 `view_file` 浏览项目结构
2. 用 `str_replace_edit` 修改代码
3. 用 `bash_exec` 运行编译/测试验证
4. 如果报错，自动查 `web_search` 找到解决方案
5. 反复迭代直到成功
6. 最后用 `task_complete` 交出完整总结

### 2.2 信息查询（免用工具）

如果你只是问一个纯知识性问题，她不会调用任何工具，直接用 AI 知识回答：

```
"Python 的 dataclass 和普通 class 有什么区别？"
"解释一下 Docker 的 overlay2 存储驱动"
```

### 2.3 必备安全约束
- 修改代码**前必须先读取原文**（防止乱改）
- 每次只改**一处**，验证通过再继续
- 危险命令（`rm -rf /`, `format` 等）自动拦截
- 联网搜索一次任务最多 5 次，单个文件最多改 10 次

## 三、23 个工具完整指南

### 第一梯队：5 个核心编程工具

#### 3.1 `view_file` — 读文件 / 列目录

**这是你用得最多的工具。** 任何代码操作的第一步。

| 参数 | 说明 | 示例 |
|------|------|------|
| `path` | 文件/文件夹的绝对路径 | `C:\project\src\main.rs` |
| `start_line` | 从第几行开始读（可选） | `0` |
| `end_line` | 读到第几行（可选） | `100` |

**怎么用：**

```
"看一下项目的目录结构"             → view_file("C:\my-project", 0, 50)
"读取app.js 的前 100 行"           → view_file("C:\my-project\app.js", 0, 100)
"查看config.rs 第 200 到 350 行"  → view_file("C:\my-project\config.rs", 200, 350)
```

#### 3.2 `str_replace_edit` — 精确修改代码

**修改代码的唯 一工具**。用"旧字符串 → 新字符串"的替换方式，确保精准。

| 参数 | 说明 |
|------|------|
| `path` | 要改的文件路径 |
| `old_str` | 要被替换的原代码（必须和文件里一模一样） |
| `new_str` | 替换后的新代码 |
| `create_if_missing` | 文件不存在就新建（默认 true） |

**怎么用：**

```
"把main.rs里的端口8080改成3000"   → old_str="8080", new_str="3000"
"删除utils.js 中第15-20行的注释"   → old_str="第15-20行原文", new_str=""
"创建新的.env 文件"                → old_str="", new_str="文件内容", create_if_missing=true
```

**重要：**
- `old_str` 必须和文件原文**逐字完全匹配**，包括空格和缩进
- 改之前**必须先用 view_file 读取**那一部分原文
- 一次只改一处逻辑相关的代码

#### 3.3 `bash_exec` — 执行命令

**运行编译、测试、安装、git 操作等所有命令行任务。**

| 参数 | 说明 | 示例 |
|------|------|------|
| `command` | 要执行的命令 | `cargo build --release` |
| `cwd` | 工作目录（可选） | `C:\my-project` |
| `timeout` | 超时秒数（默认 30） | `60` |

**怎么用：**

```
"编译这个 Rust 项目"              → cargo build
"跑一下测试"                     → cargo test
"安装 express 依赖"              → npm install express
"查看 git 状态"                  → git status
"提交并推送代码"                  → git add . && git commit -m "msg" && git push
```

**自动拦截的危险命令：** `rm -rf /`, `:(){ :|:& };:`, `mkfs.*`, `dd if=`, `format`, `shutdown`, `reboot`, `del /f /s C:\`

#### 3.4 `web_search` — 联网搜索

**当本地代码搞不定时，上网查。**

| 参数 | 说明 | 示例 |
|------|------|------|
| `query` | 搜索关键词 | `"react useEffect cleanup 内存泄漏"` |

**怎么用：**

```
"React useEffect return cleanup 的正确用法是什么？"
"nginx 502 bad gateway 怎么排查？"
"rust sqlx::Error::PoolTimedOut 是什么原因？"
```

**限制：** 一次任务最多搜 5 次。如果达到上限，请基于已有信息继续推进。

#### 3.5 `task_complete` — 任务完成 / 交卷

**所有步骤完成后，调用它提交最终总结。** 调用后本轮任务结束。

| 参数 | 说明 |
|------|------|
| `summary` | 最终总结：改了哪些文件、运行结果、注意事项 |

## 四、记忆系统：让江秋月越来越懂你

### 4.1 两种检索模式

| 模式 | 原理 | 适用场景 |
|------|------|----------|
| **向量模式**（默认） | FastEmbed 本地 ONNX 模型语义编码 | 自然语言检索，理解你的意思 |
| **SQL 模式**（降级） | SQL LIKE 关键词模糊匹配 | 向量模型加载失败时的后备方案 |

### 4.2 什么时候用记忆？

```
对话开始时：  "上次我们聊到的那个端口号是多少？"  ← search_memory
对话过程中：  "记住 React 生产环境要关闭 StrictMode"  ← add_memory
长期积累后：  你的偏好、历史决策、重要配置都会自动被记住
```

### 4.3 主界面操作

在左侧面板**记忆**标签中：
- 查看已保存的记忆列表
- 一键切换向量 / SQL 模式
- 如果向量模式加载失败，会自动降级并弹窗提示

## 五、Provider 切换：换 AI 大脑

### 5.1 支持的后端

| Provider | 默认模型 | 说明 |
|----------|----------|------|
| DeepSeek | `deepseek-v4-flash` | 默认，性价比高 |
| OpenAI | `gpt-4o` | 需要 OpenAI API Key |
| 自定义兼容 | 任意 | 任何 OpenAI 兼容 API |

### 5.2 怎么切换？

**方式一：可视化窗口**（第一次启动自动弹出，或点设置按钮）

填写 API Key、接口地址、模型名称，点保存即可。

**方式二：`.env` 文件**（开发者用）

```bash
# 切换到 OpenAI
LLM_API_KEY=sk-your-openai-key
LLM_PROVIDER=openai
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o
```

**方式三：环境变量**

直接设系统环境变量，优先级最低。

### 5.3 配置优先级

```
运行时配置 (runtime_config.json)  >  .env 文件  >  系统环境变量
```

## 六、安全机制

### 6.1 危险命令黑名单

以下命令会被**自动拦截**，不会执行：

```
rm -rf /          # 删除整个根目录
format C:         # 格式化磁盘
:(){ :|:& };:    # Fork 炸弹
mkfs.*            # 创建文件系统
dd if=/dev/zero   # 覆盖磁盘
shutdown /s       # 关机
del /f /s C:\     # Windows 递归删除 C 盘
```

执行后会在聊天窗口显示警告。

### 6.2 Docker 沙箱（可选）

如果系统安装了 Docker，所有命令会在容器中执行：
- 内存限制、网络限制、只读挂载
- Docker 不可用时**自动降级**为本地执行，不影响使用

## 八、常见问题

### Q1：她改了我没让她改的代码怎么办？
用 `git restore` 回退，然后在下次指令中说清楚只改哪个文件。

### Q2：联网搜索挂了 / 次数用完了？
告诉江秋月"联网搜索次数已用完，请基于已有信息继续"。

### Q3：向量记忆不工作？
左侧面板手动切换到 SQL 模式即可，功能不受影响。

### Q4：换了 OpenAI 后感觉回答变了？
不同模型的回答风格确实不同。可以尝试调整 `.env` 中的 temperature 参数。

### Q5：Docker 不想用？
不影响。没有 Docker 会自动降级为本地执行。

### Q6：任务执行到一半卡住了？
江秋月单次任务最多 30 轮迭代。如果超了会自动停止，你可以重新描述需求继续。

## 九、一句话总结

**把江秋月当作一个能读代码、能写代码、能运行命令、能上网查资料的自动化助手。** 你只需要说"我想要什么结果"，她会自己想办法一步步完成。

试试看最经典的第一条指令：

> "帮我分析一下整个项目的结构，列出来有哪些模块和文件"

然后看着她是如何用 `view_file` 自动遍历项目目录的。
